# Критерии оценки проекта

## 1. Сбор данных (3–10 баллов)

### a) Обязательная часть
- [ ] **i. Функционал сбора потоковых данных:** разделение исходного набора на батчи и эмуляция потока **(1 балл)**
- [ ] **ii. Разработка хранилища сырых данных:** файловая система **(1 балл)** или БД **(2 балла)**
- [ ] **iii. Расчет метапараметров:** **(1-2 балла)**

### b) Дополнительные баллы
- [ ] **i. Создание конфигурационного файла** (.py / YAML/JSON/TOML/XML) с гиперпараметрами сбора **(1 балл)**
- [ ] **ii. Интеграция с несколькими источниками данных** **(2 балла)**
- [ ] **iii. Система логирования + обработка ошибок** при сборе данных **(1-2 балла)**

## 2. Анализ данных (3–13 баллов)

### a) Обязательная часть
- [ ] **i. Оценка и хранение показателей качества данных** (data quality) **(1-2 балла)**
- [ ] **ii. Ассоциативные правила (Apriori / FP-tree):** ≥5 правил для проверки корректности или обогащения признаков **(1-3 балла)**
- [ ] **iii. Базовая очистка данных** на основе порогов допустимого качества **(1 балл)**

### b) Дополнительные баллы
- [ ] **i. Автоматический EDA** **(1-2 балла)**
- [ ] **ii. Feature Engineering** **(1-2 балла)**
- [ ] **iii. Генерация отчетов о качестве данных** **(1 балл)**
- [ ] **iv. Мониторинг и обработка data drift** **(1-2 балла)**

## 3. Подготовка данных (0–5 баллов)

### a) Обязательная часть (зависит от модели ML)
- [ ] **i. Обработка пропусков** **(0-1 балл)**
- [ ] **ii. Обработка категориальных переменных** **(0-1 балл)**
- [ ] **iii. Обработка числовых переменных** **(0-1 балл)**

### b) Дополнительные баллы
- [ ] **i. Несколько вариантов предобработки** + перебор при поиске лучшей модели **(1-2 балла)**

## 4. Обучение/дообучение модели (1–5 баллов)

### a) Обязательная часть
- [ ] **i. Построение ≥2 моделей:** деревья решений / ансамбли на деревьях **и** нейронные сети **(1 балл)**

### b) Дополнительные баллы
- [ ] **i. Дообучение предыдущей модели** (без обучения с нуля) **(1-2 балла)**
- [ ] **ii. Разработка моделей с разной устойчивостью** к входным данным **(1-2 балла)**

## 5. Валидация модели (2–10 баллов)

### a) Обязательная часть
- [ ] **i. Оценка качества модели:** hold-out / CV / TimeSeriesCV **(1-3 балла)**
- [ ] **ii. Хранилище версий моделей + контроль качества** **(1-2 балла)**

### b) Дополнительные баллы
- [ ] **i. Интерпретация прогнозов:** визуализация дерева, коэффициенты LR, ближайшие соседи, LIME, SHAP **(1-3 балла)**
- [ ] **ii. Мониторинг и обработка model drift** **(1-2 балла)**

## 6. Обслуживание модели (1–6 баллов)

### a) Обязательная часть
- [ ] **i. Выбор и упаковка (сериализация) финальной модели** (одной или нескольких) **(1-2 балла)**

### b) Дополнительные баллы
- [ ] **i. Мониторинг производительности** (время применения / память) **(1-2 балла)**
- [ ] **ii. Гибкий прогноз** (выбор модели при разреженных данных / аномалиях) **(1-2 балла)**

## 7. Управление программой (3–11 баллов)

### a) Обязательная часть
- [ ] **i. Скрипт управления конвейером** (Inference, Update, Summary) **(2 балла)**
- [ ] **ii. Документация:** README + requirements.txt **(1 балл)**

### b) Дополнительные баллы
- [ ] **i. Расширенный отчёт (dashboard)** о работе системы **(1-2 балла)**
- [ ] **ii. Конфигурационный файл параметров** всех компонентов (батч, пропуски, валидация и т.д.) **(1-2 балла)**
- [ ] **iii. Meta Learning модель:** оценка гиперпараметров + динамика метрик **(2 балла)**
- [ ] **iv. Архитектура на основе паттернов проектирования** (например, MVC) **(1-2 балла)**
