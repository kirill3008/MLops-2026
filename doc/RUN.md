## Документация к скрипту "run.py"

Главный скрипт управления MLOps пайплайном. Предоставляет интерфейс командной строки для запуска различных режимов работы: обучение моделей, инференс, обновление моделей и генерация отчётов.

---

### Использование

```bash
python run.py -mode "pipeline" -initial_batches 3 -max_batches 20
python run.py -mode "inference" -file "./data.csv"
python run.py -mode "update" -batch_limit 5
python run.py -mode "summary"
```

---

### Аргументы командной строки

#### **`-mode`** (обязательный)
- **Допустимые значения**: `pipeline`, `inference`, `update`, `summary`
- **Описание**: Режим работы пайплайна.

#### **`-file`** (для режима `inference`)
- **Тип**: `str`
- **Описание**: Путь к CSV-файлу для предсказаний.

#### **`-initial_batches`** (для режима `pipeline`)
- **Тип**: `int`
- **По умолчанию**: `3`
- **Описание**: Количество батчей для начального обучения модели.

#### **`-update_every`** (для режима `pipeline`)
- **Тип**: `int`
- **По умолчанию**: `5`
- **Описание**: Периодичность обновления модели (каждые N батчей).

#### **`-max_batches`** (для режима `pipeline`)
- **Тип**: `int`
- **По умолчанию**: `20`
- **Описание**: Максимальное количество батчей для обработки.

#### **`-batch_limit`** (для режима `update`)
- **Тип**: `int`
- **По умолчанию**: `5`
- **Описание**: Лимит батчей для режима обновления.

### Режимы работы

#### 1. **Режим `inference`**

```python
def inference_mode(self, file_path: str):
```

**Описание:**
- Загружает данные из CSV-файла.
- Применяет предобработку к данным.
- Делает предсказания с помощью лучшей модели из реестра.
- Сохраняет результаты в файл `inference_result_YYYYMMDD_HHMMSS.csv`.

**Этапы:**
1. Проверка существования входного файла.
2. Загрузка лучшей модели из реестра (`_load_best_model()`).
3. Чтение и предобработка данных через `DataPreprocessor`.
4. Выполнение предсказаний.
5. Сохранение результатов с исходными данными и колонкой `predict`.

**Возвращает**: Путь к файлу с результатами или `None` при ошибке.

---

#### 2. **Режим `update`**

```python
def update_mode(self, batch_limit: int = 5):
```

**Описание:**
- Загружает батчи данных из потока.
- Проверяет качество данных (DQ).
- Инкрементально обновляет модель на новых данных.

**Этапы:**
1. Загрузка лучшей модели.
2. Создание потока данных (`DataStream`).
3. Для каждого батча:
   - Оценка качества данных через `evaluate_reference_rules_on_batch()`.
   - Предобработка данных.
   - Инкрементальное обновление модели (если есть метки).
4. Обновление лучшей модели в реестре.

**Возвращает**: `True` если обработан хотя бы один батч, `False` при ошибке.

---

#### 3. **Режим `summary`**

```python
def summary_mode(self):
```

**Описание:**
- Собирает метрики со всех компонентов пайплайна.
- Генерирует JSON-отчёт и визуальный дашборд.

**Собираемые метрики:**
- **Model metrics**: метрики лучшей модели, количество версий в реестре.
- **Data quality**: пропуски, дубликаты, невалидные значения.
- **Data drift**: обнаруженные дрифты, затронутые признаки, уверенность.
- **Performance**: accuracy, F1, время инференса по батчам.
- **System info**: версия Python, использование памяти, количество CPU.

**Выходные файлы:**
- `summary_report_YYYYMMDD_HHMMSS.json` — JSON с метриками.
- `summary_report_YYYYMMDD_HHMMSS_dashboard.png` — визуализация (6 графиков).

---

#### 4. **Режим `pipeline`**

```python
def pipeline_mode(self, initial_batches: int = 3, update_every: int = 5, max_batches: int = 20):
```

**Описание:**
- Полный цикл MLOps: начальное обучение → инкрементальная обработка → периодическое обновление.

**Этапы:**

##### **Фаза 1: Начальное обучение**
1. Загружает `initial_batches` батчей из потока данных.
2. Проверяет качество данных для каждого батча.
3. Объединяет батчи и обучает модели с нуля (`ModelPipeline.run()`).
4. Сохраняет лучшую модель в реестр.

##### **Фаза 2: Инкрементальная обработка**
1. Обрабатывает оставшиеся батчи до `max_batches`.
2. Для каждого батча:
   - Проверка качества данных.
   - Предсказания текущей моделью.
   - Вычисление метрик (accuracy, F1) при наличии меток.
   - Сохранение метрик производительности.
3. Каждые `update_every` батчей:
   - Инкрементальное обновление модели на новых данных.
   - Регистрация обновлённой модели в реестре.

**Выходные данные:**
- Файл `pipeline_performance_YYYYMMDD_HHMMSS.csv` с метриками по батчам.
- Обновлённые модели в реестре.

**Возвращает**: `True` при успешном выполнении, `False` при ошибке.

### Вспомогательные методы

#### **`_load_best_model()`**
```python
def _load_best_model(self):
```
- Загружает лучшую модель из файла `model_registry/best_model.json`.
- Десериализует модель через `joblib.load()`.

---

#### **`_update_best_model()`**
```python
def _update_best_model(self):
```
- Обновляет атрибут `self.best_model` из пайплайна или из реестра.

---

#### **Методы сбора метрик**

##### **`_collect_model_metrics()`**
- Загружает метрики из `best_model.json` и `models_metadata.json`.
- Возвращает accuracy, F1, количество версий моделей.

##### **`_collect_data_quality()`**
- Читает последние JSON-файлы из директории `artifacts/dq/`.
- Возвращает `missing_total_ratio`, `duplicate_ratio`, `invalid_ratio`.

##### **`_collect_drift_metrics()`**
- Читает отчёты о дрифте из `artifacts/rules/`.
- Возвращает тип дрифта, уверенность, затронутые признаки.

##### **`_collect_performance_metrics()`**
- Читает CSV-файлы производительности (`pipeline_performance_*.csv`).
- Вычисляет средние значения accuracy, F1, время инференса.

##### **`_collect_system_info()`**
- Собирает информацию о системе: версия Python, платформа, использование памяти (через `psutil`).

---

#### **`_generate_comprehensive_dashboard(summary_data, report_path)`**

Создаёт визуальный дашборд с 6 графиками:
1. **Model Performance Over Batches**: линейный график accuracy и F1 по батчам.
2. **Current Data Quality Metrics**: барчарт пропусков, дубликатов, невалидных значений.
3. **Memory Usage**: круговая диаграмма использования памяти.
4. **Data Drift Status**: барчарт обнаруженных типов дрифта.
5. **Model Performance Evolution**: заглушка для истории выбора моделей.
6. **System Performance Summary**: горизонтальный барчарт сводных метрик.

Сохраняет дашборд в PNG-файл.

### Функция `main()`

```python
def main():
    parser = argparse.ArgumentParser(description='MLOps Pipeline Controller')
    # ... добавление аргументов ...
    args = parser.parse_args()
    
    pipeline = MLOpsPipeline()
    
    # Выбор режима и запуск
    if args.mode == 'inference':
        result = pipeline.inference_mode(args.file)
    elif args.mode == 'summary':
        report_path = pipeline.summary_mode()
    elif args.mode == 'update':
        success = pipeline.update_mode(batch_limit=args.batch_limit)
    elif args.mode == 'pipeline':
        success = pipeline.pipeline_mode(...)
```

**Описание:**
- Парсит аргументы командной строки.
- Создаёт экземпляр `MLOpsPipeline`.
- Вызывает соответствующий метод на основе режима.
- Обрабатывает ошибки и завершает программу с кодом `1` при неудаче.

---

### Структура выполнения

1. **Парсинг аргументов** через `argparse`.
2. **Инициализация пайплайна** (`MLOpsPipeline()`).
3. **Выполнение выбранного режима**:
   - `inference`: предсказания на новых данных.
   - `update`: инкрементальное обновление модели.
   - `summary`: генерация отчёта и дашборда.
   - `pipeline`: полный цикл обучения и обновления.
4. **Логирование результатов** в файл `pipeline.log` и stdout.
